Cloner et récuperer les fichiers d'un repository

(A partir d'ici travailler à plusieurs pour pouvoir faire l'exercise, inviter vous dans vos repository si cela n'est pas fait via github)

Cloner et récuperer la même chose ?

Cloner un repository et récuperer un repository sont très diffèrent voici leur difference :

git clone git pull
1. Utilisé pour configurer un référentiel local. 1. Utilisé pour synchroniser les référentiels distants et locaux.
2. Faites une copie de votre référentiel distant et enregistrez-la sur votre machine locale. 2. Récupérez et fusionnez les dernières modifications d’un référentiel distant vers votre référentiel local.
3. Utilisé une fois dans un projet. 3. Utilisé plusieurs fois dans un projet.

Comment mettre on pratique ?

Pour cloner un repository il faut faire cette commande 

git clone <repository> [<branche>]

La branche est nécessaire seulement si vous voulez prendre une branche alternative à la principal

Pour récuperer / pull il faut faire cette commande

git pull [<options>] [<repository>]

Les options sont supplémentaire pour faire des choses plus précise et le repository est nécessaire seulement si on à jamais pull

Si vous voulez juste faire pull il faut être dans le dossier ou le fichier git est présent
